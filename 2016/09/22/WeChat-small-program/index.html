关于微信小程序

发表于 2016-09-22   |   分类于 互联网   |  

收集一些信息，不断更新，做准备吧。

1.建议有心的朋友可以现在就开始收集各类小众，低频次，但是刚需的小应用需求，甚至可以自己就开始设计开发，等到微信应用号一推出，肯定能够占到先机，赚到自己的第一桶金。

2.如果了解过微信 JS SDK 的伙伴一定觉得：卧槽，微信团队真够替前端考虑的，又特么是这套差不多的东西。

有前端 HC 的赶紧招人吧，预计下半年能看入眼的前端会更难招了。

目前这个开发套路也没啥好多聊的(废话，靠YY劳资也编不下去了呀)，我想和感兴趣的前端小伙伴们说：先把技术储备准备起来吧，以下内容请自行搜索：

  • 微信JS SDK
  • WeUI
  • Vux

技术就先扯到这。

3.在阅读完产品文档并做完产品线框图后，我越来越相信微信小程序，未来将掀起一波潮流。小程序的体验和原生 app 几乎没有差异。

然而，我们公司的开发能力并不足够，我们想利用十一长假，做一次封闭开发，需要外部力量的支持，如果你想利用十一长假赚一点零用钱且比绝大多数人更早地参与微信应用号（小程序）开发，这是一次绝佳机会。

兼职时间： 10.1~10.7
报酬：不同岗位日薪 500~1000 元。
需要的人员：

  1. Python 开发，最好熟悉 Django ，必须有写过 API 的经验
  2. 前端工程师，必须熟悉 Javascript 和有过微信服务号开发经验，CSS 和 HTML 也是必须熟练的，最好有 react 开发经验。
  3. UI 设计师，喜欢扁平化设计，且有移动网页设计经验的设计师

兼职地点：北京朝阳区常营。

有兴趣的同学，请跟我联系： jason[AT]kenengba.com

4.有了「小程序」是不是大家都要一窝蜂的去做前端呢？没必要，无论前端多么牛逼，总要云端技术支持才行，否则就是无源之水，无本之木。所以，Java、PHP、Go、Lua 都是很好的技术栈，该写什么继续写什么吧。退一步说，如果大家一窝蜂去搞前端了，你留在后端领域，还是稀缺物种了呢。

5.未来的两三年里呢？

如果你有个好想法，直接做就是了。 如果你现在不会 javascript，买五六本本书来看（比如原版书三十美元一本），上个很贵的培训班（哪怕比如五万块两个月）…… 搞定之后自己动手，肯定比招一票人每个月花六七位数的钱才能养活划算的多！

也就是说，通过学习和实践，将有大量的个体用“十万人民币”就可以开始“互联网创业”了，你说是不是“史上成本最低”？

起码以下三类人会占大便宜：

  • 真正了解刚需的产品经理
  • 真正了解渠道的市场推广
  • 真正受民众欢迎的写作者

这些人若是竟然“胆敢下手学会 javascript，开始做小程序开发”，若是他们都不能财富自由的话，天理何在！
